## General Requirements

1. The system must manage Permits-To-Work (PTWs).

2. The information required must include but not limited to: **date, time, requester, assessor, approver, work location, description, pictures, checkbox for questions and answers** (if applicable).

3. The system should have the ability to **pinpoint and capture the GPS location** of the mobile device where work is done on site.

4. Users must be able to **pre-set workflows** to automate processes. These processes include:
   - Recording observations
   - Acknowledging observations
   - Submitting closure report
   - Verifying report
   - Revoking e-PTW
   - Closing the permit on mobile devices

   The workflows must **not be skipped** (i.e. from start to close-out must follow the full flow).

5. The e-PTW must ensure that **only authorised persons can assess and approve** the permit to work. There must be safeguards to disallow falsification.

6. The e-PTW must be able to capture data such as: **text, date, multiple photos per inspection, comments, timestamp of data input or changes, and identity of the user** for audit purposes.

7. **Site drawings, photos and supporting documents** should be able to be attached to the e-PTW form and be electronically routed to the authorised person via their mobile devices. **Notification must be sent** to the authorised person once submission of the form is done.

8. User must be able to **set a deadline for the Permit expiry**. The system must trigger **automated alerts** to relevant supervisors to follow up on cases close to expiry in accordance to the pre-set workflow.

9. The e-PTW application must allow the **authorised person to revoke an issued permit** to work.

10. The information must be able to be **viewed, validated and approved by the stipulated deadline**.

11. The **web interface dashboard** of the e-PTW system must show permits that are being reviewed.

12. Each e-PTW must have its own **unique identification number**.

13. **Default language** of the e-PTW must be in **English**.

## System Performance

14. The e-PTW application must be able to **run on all major mobile platform devices** such as Android and iOS, including any new software updates.

## Key Points from MOM Circular (Annex A Summary)

- **Mandatory from 1 April 2024**: ePTW adoption is mandatory for all public sector construction and construction-related projects valued at **S$3 million and above**.
- **Record retention**: Clients must keep records for a **minimum of 5 years**.
- **Roles**: Requester (applicant) → Assessor (verifier) → Approver
- **Real-time monitoring**: Status of permits must be monitorable in real-time from mobile devices.
